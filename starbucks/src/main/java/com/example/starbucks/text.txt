    웹 서버 에는

*컨트롤러(controller) : 경로 지정

*서비스(Service) : 계약[약속]
-> 클래스 하나 더 만들어서 service implement 한 후 재정의

*레포지토리(repository)   [Java <-> ORM(JPA) <-> DB(SQL문)]
-> DB에서 데이터 가져오는 역할. (JpaRepository 를 extends함)
    리포지터리는 엔티티에 의해 생성된 데이터베이스 테이블에 접근하는 메서드들
    (예: findAll, save 등)을 사용하기 위한 인터페이스이다.


*dto : Controller, Service, Repository 등

       계층 간 데이터 교환을 할 때의 Entity로 통신하는 것은

       보안에 좋지 못하기 때문에 DTO를 사용한다.

*status : 상태를 나타내는 상수 정의 및 그에 따른 코드 설명



순서
1. repository에서 입력
2. Service 선언 후 serviceimplement 에서 재정의
3. controller에서 경로 지정 및 조건문 작성
